# API endpoint methods (hosts, enrollment code, downloads)

Implement the typed endpoint methods `dn-tool` actually calls, with the correct HTTP method, **API version**, path, request body, and envelope unwrapping. Mind the version split: create/get/list hosts are **v2**; host delete is **v1 only** (no v2 successor); downloads is `v1` (API reference §3).

## Public interface

```go
// internal/api
func (c *Client) CreateHostAndEnrollmentCode(ctx context.Context, req CreateHostRequest) (*HostAndCode, error)
    // POST /v2/host-and-enrollment-code  (returns host + single-use enrollment code)
func (c *Client) ListHosts(ctx context.Context, networkID string) ([]Host, error)
    // GET  /v2/hosts?filter.networkID=…  (paginated via API.pagination; match name client-side)
func (c *Client) DeleteHost(ctx context.Context, hostID string) error
    // DELETE /v1/hosts/{hostID}   (v1 ONLY)
func (c *Client) ListDownloads(ctx context.Context) (*Downloads, error)
    // GET /v1/downloads  (used by INST.resolve)
```

`CreateHostRequest` carries name, networkID, roleID, tun/network name, optional static IP, tags, and the lighthouse/relay/static-address/listen-port fields ([LH.fields](dt-wleh.md)). Model the `Host` object per API reference §5 (need at least `id`, `name`).

## Behaviors (TDD order)

1. **CreateHostAndEnrollmentCode** — `POST /v2/host-and-enrollment-code` with the expected JSON body; returns host + code (the code is handled in-memory by enroll, never logged).
2. **ListHosts** — `GET /v2/hosts` with `filter.networkID`; aggregates pages (uses API.pagination); returns `[]Host` with ids/names.
3. **DeleteHost** — `DELETE /v1/hosts/{id}`; 2xx → nil; **404 → nil** (idempotent already-absent, consumed by [UNE.delete](dt-2t72.md)); other 4xx → `*APIError`.
4. **ListDownloads** — `GET /v1/downloads`; returns the downloads object for [INST.resolve](dt-hts2.md).
5. **Correct versions/paths** — assert each method hits the documented v1/v2 path and method (regression guard against version drift).

## Test strategy

`httptest.Server` asserting method + path + decoded request body per endpoint; return canned envelopes. Cover the 404-as-success branch for DeleteHost explicitly.

## Acceptance

- All four methods use the correct version/method/path and unwrap the envelope.
- DeleteHost treats 404 as success; other failures return `*APIError`.
