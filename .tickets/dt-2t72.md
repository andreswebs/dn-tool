# Unenroll delete + local removal (2xx/404)

Implement the unenroll happy path: read the host_id (UNE.hostid), `DELETE` the remote record, and on a `2xx` or already-absent `404` remove the local configuration directory for the network. Bound the whole operation by `DN_API_TIMEOUT` (~10s), which must be shorter than the surrounding service stop timeout.

## Public interface

```go
// internal/unenroll (or internal/enroll)
func Unenroll(ctx context.Context, cfg *config.Config, deps Deps) (output.Result, error)
//   ctx already bounded by DN_API_TIMEOUT at the command layer
//   read host_id -> api.DeleteHost(ctx, id) -> on 2xx/404 remove <root>/<network>/
```

`api.DeleteHost` already maps `404 → nil` (idempotent, [API.endpoints](dt-255n.md)). Local removal is `os.RemoveAll` on the network config dir, only after a successful/idempotent delete.

## Behaviors (TDD order)

1. **Delete 2xx → remove local** — mock delete succeeds → local config dir removed; `Result{Action:"unenroll", Changed:true}`.
2. **Delete 404 → treated as success** — already-absent remote → local removed, no error.
3. **Not enrolled → clear error** — no local config (UNE.hostid not-enrolled) → fail clearly, nothing removed.
4. **Bounded by deadline** — the operation runs under the `DN_API_TIMEOUT` context (the failure-retention behavior is [UNE.invariant](dt-f7nx.md)).

## Test strategy

`t.TempDir()` config root with a fixture `dnclient.yml`; mock api client scripting delete 2xx / 404 / error. Assert local dir presence/absence and `Result`.

## Acceptance

- 2xx and 404 both lead to local removal; not-enrolled fails clearly; bounded by `DN_API_TIMEOUT`.
