# Assert-changed no-op exits with status 2

Implement the `--assert-changed` semantics: when the flag is set and a command makes **no change**, exit with the distinct non-error, non-zero status **2**. Reads the `Changed` field of the command `Result` ([OUT.result](dt-ccmn.md)).

## Public interface

```go
// applied in the command wrapper, building on EXIT.map (dt-icq8)
//   if assertChanged && !result.Changed { return ExitError(nil, CodeNoOpAssert /* 2 */) }
```

`2` is reserved exclusively for the assert-changed no-op — never reused for failures.

## Canonical mapping (shared with the epic)

| Code | Meaning |
| ---- | ------- |
| 0 | Success (changed, or no-op without `--assert-changed`). |
| 1 | Failure. |
| 2 | No-op while `--assert-changed` is set. |

## Behaviors (TDD order)

1. **No-op + `--assert-changed` → 2** — `Changed=false` and flag set → exit 2.
2. **Change + `--assert-changed` → 0** — `Changed=true` and flag set → exit 0.
3. **No-op without the flag → 0** — `Changed=false`, flag unset → exit 0.
4. **Failure still → 1 even with the flag** — an errored command exits 1 regardless of `--assert-changed` (2 is not for failures).

## Test strategy

Drive commands returning `Result{Changed:…}` and an error/nil through the app with/without `--assert-changed`; assert the resolved exit code per the table.

## Acceptance

- The four cells above hold; `2` never collides with failure.
