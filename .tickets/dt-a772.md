# dnclient Enroller interface + exec + mock

Define the `dnclient` subprocess abstraction so enroll (and later `run`) are testable without the proprietary binary, plus the real exec implementation. Surfacing a non-zero exit from `dnclient enroll` as a clear error (Q24 → A: interface for the subprocess).

## Public interface

```go
// internal/dnclient
type Client interface {
    Enroll(ctx context.Context, code string) error  // exec: dnclient enroll -code <code> ...
    Run(ctx context.Context, args ...string) error   // exec: dnclient run ... (used by RUN.compose)
}

type execClient struct{ binPath string } // real impl: exec.CommandContext(binPath, ...)
func NewExecClient(binPath string) Client
```

The enrollment `code` is passed in-memory only — never logged (OUT.redact). The real impl uses `exec.CommandContext` against the installed binary path (from [INST.place](dt-svmu.md)); stderr/stdout wiring should avoid leaking the code.

## Behaviors (TDD order)

1. **Interface drives enroll** — a mock `Client` records the `code` it was called with; enroll passes the obtained code through.
2. **Non-zero exit surfaced** — mock returns an error → enroll fails and surfaces it (don't swallow).
3. **Context cancellation kills the subprocess** — `Run`/`Enroll` honor `ctx` (real impl via `CommandContext`).
4. **Code not logged** — exercising the exec path with a fake code produces no log line containing it (ties to OUT.redact).

## Test strategy

Unit-test enroll against a mock `Client`. For the real `execClient`, a focused test can exec a tiny helper script (or `os.Args[0]`-as-fake-binary trick) to assert arg passing and exit-code propagation — keep it minimal and Linux-guarded.

## Acceptance

- Enroll/run depend on the `Client` interface (mockable); real impl execs the binary, honors `ctx`, surfaces non-zero exits, and never logs the code.

Added internal/dnclient/client.go: Client interface (Enroll/Run) + execClient real impl via exec.CommandContext, plus client_test.go covering all 4 behaviors.

KEY DECISIONS:
- Deviated from the ticket's literal Enroll(ctx, code string) → Enroll(ctx, networkName, code string), matching upstream 'dnclient enroll -name <network> -code <code>'. The -name is essential: it drives the per-network config dir /etc/defined/<network>/dnclient.yml that ConfigExists/ReadHostID (and unenroll) depend on. Consistent with the project's 'trust the real contract over the toy signature' pattern (dt-brug/dt-hts2/dt-4h21). NewExecClient(binPath) stays stateless per the ticket; network name is a per-call arg.
- Run(ctx, args...) prepends the 'run' subcommand; caller passes -server/-name (dt-flal consumes this).
- child stdout AND stderr wired to os.Stderr (stdout reserved for the JSON result contract §2.8); args never logged (only action+bin at debug) so the code can't leak via our logs or the error (exec.ExitError is 'exit status N', no args). Covers behavior 4 / SEC5.
- Did NOT wire Client into enroll.Deps — that is dt-pe29's job (dt-brug deferred the dnclient field to the cell that runs enroll). Scope held by omission.

TEST NOTE: the ctx-cancellation test uses 'exec sleep 30' (not bare 'sleep 30') in the fake binary. A bare sleep is an orphaned grandchild that inherits the test's stderr fd and holds the go-test pipe open for 30s after sh is killed; 'exec' makes sh replace itself so CommandContext kills the one direct process — also models the real single-direct-child case.
